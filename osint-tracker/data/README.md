# Data Directory Guide

This directory holds all raw API responses and processed pipeline outputs.

```
data/
├── raw/                        ← Raw API responses, cached automatically
│   ├── sample_ioda_iran_20260228.json        (internet outage data)
│   ├── sample_acled_iran_20260228.json       (conflict events)
│   ├── sample_newsapi_iran_20260228.json     (news volume signals)
│   └── <md5_hash>.diskcache                 (auto-generated by diskcache)
│
└── processed/                  ← Final pipeline outputs
    ├── sample_pipeline_output_iran_20260228.json   (full scored report)
    └── iran_20260228_2218.html                     (interactive map)
```

---

## Raw Data (`data/raw/`)

Raw files are **automatically cached** by the pipeline using `diskcache`.
You never need to create them manually — they appear after the first run.

The sample JSON files in this folder show exactly what each API returns
so you can understand the data structure before running anything live.

### IODA Response Structure
```json
{
  "data": {
    "bgp":          { "values": [...] },   ← BGP routing health (array of ints)
    "ping-slash24": { "values": [...] },   ← Active probe response rate
    "ucsd-nt":      { "values": [...] }    ← Darknet telescope traffic
  }
}
```
Values drop sharply when connectivity collapses. The pipeline computes
what percentage of baseline is lost → outage_score (0.0–1.0).

### ACLED Response Structure
```json
{
  "data": [
    {
      "event_date":    "2026-02-28",
      "event_type":    "Explosions/Remote violence",
      "sub_event_type":"Air/drone strike",
      "location":      "Tehran",
      "latitude":      "35.6892",
      "longitude":     "51.3890",
      "fatalities":    "0",
      "notes":         "Airstrike reported near IRGC HQ..."
    }
  ]
}
```

### NewsAPI Response Structure
```json
{
  "totalResults": 847,
  "articles": [
    {
      "source":      { "name": "Reuters" },
      "title":       "...",
      "publishedAt": "2026-02-28T06:14:00Z"
    }
  ]
}
```

## Processed Data (`data/processed/`)

Processed files are the **output** of the pipeline — after collection,
correlation, anomaly detection, and scoring.

### Pipeline Output JSON
Contains the full scored report:
- `scores.composite` → 0–10 activity score
- `signals.*` → per-layer anomaly details
- `signals.acled.hotspots` → top conflict locations
- `signals.aircraft.isr_spotted` → notable aircraft

### HTML Map
A self-contained interactive map file. Open directly in any browser:
```bash
open data/processed/iran_20260228_2218.html
```
No server needed — everything is bundled inside the single file.

---

## Cache Management

Clear the raw cache to force fresh API calls:
```bash
rm -rf data/raw/
```

Clear processed outputs:
```bash
rm -rf data/processed/
```

Both directories are in `.gitignore` — data files are never committed.
